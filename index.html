企业 AI 视频营销引擎

07 / 22
LIBRARY 01
PRODUCT ASSETS
库一

产品素材库

先回答卖什么、给谁看、凭什么相信。

IMAGE
产品图统一主图、细节图、使用图可直接调用
PROOF
卖点有证据参数、案例、对比、禁用词分开管理
SCENE
场景可复用门店、私域、销售跟进、投放测试
08 / 22
LIBRARY 02
REFERENCE VIDEOS
库二

参考视频库

不是抄爆款,是复用被市场验证过的成交表达。

HOOK
借开头客户为什么停下来,前三秒怎么抓住
RHYTHM
借节奏哪里加速、停顿、给证明和对比
BOUNDARY
不借品牌不复制 IP、口号、人物和独家视觉
09 / 22
LIBRARY 03
CHARACTERS / SCENES
库三

角色场景库

让客户反复看到稳定的人和场,形成信任入口。

PERSON
固定谁来说老板、销售、虚拟代言人、员工角色
PLACE
固定在哪里门店、办公室、直播间、产品展示台
TRUST
固定怎么说语气、专业领域、禁用表达保持一致
10 / 22
LIBRARY 04
STORYBOARD / SCRIPT
库四

分镜脚本库

把一句卖点拆成客户看得懂的成交镜头。

PAIN
先说痛点让客户看到自己的问题,不是先夸产品
PROOF
再给证明用动作、对比、细节镜头证明卖点
CTA
最后收口导向咨询、到店、私信、销售跟进
11 / 22
LIBRARY 05
PROMPT MODULES
库五

提示词库

不是神奇咒语,是把营销意图翻译成模型指令。

INPUT
输入变量卖点、人群、场景、角色自动组合
SHOT
镜头变量景别、运动、光线、节奏稳定可控
RULE
品牌边界不能说什么、不能像谁、不能怎么拍
12 / 22
LIBRARY 06
EDITING / PUBLISHING
库六

剪辑发布库

AI 生成只是素材,剪辑发布才让它变成客户触点。

EDIT
统一包装字幕、节奏、音乐、转场和品牌规范
CUT
多版输出销售、私域、投放、渠道各有版本
DATA
反馈回流把播放、咨询、成交反馈回到前五个库
13 / 22
WORKFLOW REVIEW
INPUT TO CUSTOMER TOUCH
协作链路

六个库如何
从卖点走到客户触达

不是某个工具单点提效,而是每一步都能复用上一环的资产。

产品资产确定卖什么、给谁看、凭什么信
参考结构确定什么表达已经被市场验证
角色场景确定谁出现、出现在哪里
成交分镜确定先讲什么、后证明什么
模型执行把营销意图变成可控指令
多渠道分发把生成片段变成客户触点
14 / 22
OPERATING MODEL
AI DOES THE LOOP
企业用法

不是增加岗位协同
是 AI 接管内容链路

AI STEP 01
目标理解读取产品、客群、渠道和边界,判断内容主攻方向。
AI STEP 02
内容策划自动拆卖点、选参考、排脚本和分镜。
AI STEP 03
素材生成生成镜头、口播、字幕、封面和多渠道版本。
AI STEP 04
反馈迭代根据播放、咨询、转发数据,反推下一轮内容。

OUTPUT
成交触点选题、分镜、提示词、视频素材、销售跟进版本。

CASE 01
PREMIUM TEA GIFT
案例问题

某高端茶礼
视频带货卖不动

只拍礼盒像广告,客户不知道贵在哪里,销售也讲不清为什么适合商务送礼。

VALUE
价值没讲透年份、产地、陈皮、仓储和包装都没有被解释。
SCENE
场景没建立商务拜访、客户维护、节日采购没有被拍出来。
SALES
销售没弹药只能发图册和价格,缺少能转发的短视频理由。
